# README3 — دليل مفصّل وشامل باللغة العربية

هذا المستند هو إصدار مطوّل من `README3.md` يشرح بالتفصيل اللغات والتقنيات المستعملة في مشروع My Order، بنية الملفات، كيفية التشغيل المحلي، كيفية التعديل على الأجزاء الأساسية، ونصائح لتطوير أو نشر التطبيق.

## لماذا هذا الدليل؟

إذا انضم مطوّر جديد إلى المشروع أو أردت صيانة سريعة، هذا الملف يوفّر مرجعاً واضحاً ومفصّلاً يشرح أين توجد كل قطعة من الشيفرة وكيف يمكن تعديلها أو تشغيلها.

---

محتويات هذا الدليل:

- نظرة عامة على المشروع والهدف منه
- شجرة الملفات الأساسية وشرح لكل ملف
- شرح مفصّل حسب اللغة/التقنية
- إعداد قاعدة البيانات محلياً
- نقاط الـ API المهمة مع وصف مبسّط
- شرح لوحة الإدارة وكيفية تعديلها
- إعداد PWA وService Worker
- سكربتات التشغيل والتطوير المحلي
- أمثلة عملية لتعديلات شائعة (تغيير اسم الموقع، الأيقونات، إعدادات DB)
- قائمة مشاكل شائعة وحلول سريعة
- خطوات للنشر والمساهمة

---

1. نظرة عامة على المشروع

My Order هو تطبيق ويب لطلبات الطعام يهدف إلى توفير واجهة سريعة وسهلة للاستخدام للزبائن مع لوحة إدارة لإدارة الطلبات والإحصاءات. يستخدم تقنيات ويب قياسية في الواجهة (HTML/CSS/JS) وPHP+MySQL في جانب الخادم.

المبادئ التصميمية:

- فصل واجهة المستخدم عن منطق الخادم (frontend / backend).
- توفير تجربة PWA (يمكن تثبيته كـ تطبيق على الهاتف وتقديم عمل جزئي عند انقطاع الشبكة).
- سهولة القراءة والتعديل عبر ملفات مفصّلة وواضحة.

---

2. شجرة الملفات الأساسية (شرح سريع لكل ملف/مجلد)

- `index.html` — الصفحة الرئيسية: تحتوي على بنية الواجهة، عناصر التصفح، ربط ملفات CSS/JS ووسوم الـ meta.
- `style.css` — الأنماط العامة، متغيرات الألوان والتخطيط.
- `script.js` — الملف الذي يحتوي غالبية منطق واجهة المستخدم: إدارة السلة، إنشاء الطلب، إرسال الطلبات للـ API، وطباعة الفواتير.
- `manifest.json` — إعدادات PWA (الاسم، الأيقونات، ألوان الثيم، الشورتكات).
- `sw.js` — Service Worker: سياسة الكاش وتحديث الملفات للعمل في وضع عدم الاتصال.
- `register-sw.js` — سكربت بسيط لتسجيل الـ Service Worker في المتصفح.
- `api/` — مجلد يحتوي على نقاط النهاية والوظائف الخادمة (PHP):
  - `db.php` — إعداد اتصال قاعدة البيانات ووظائف مساعدة.
  - `order.php` — استلام طلب جديد.
  - `orders.php` — استرجاع الطلبات.
  - `update_order.php` — تحديث حالة طلب.
- `admin/` — لوحة الإدارة: صفحات PHP لإدارة الطلبات، الإحصاءات وتعديل البيانات.
- `schema.sql` — ملف SQL يحتوي على تعريف الجداول المطلوبة (orders، activity_logs، user_sessions، ...).
- `icons/` — مجلد الأيقونات والـ apple-touch-icons.
- أدوات التشغيل: `START_SERVER.bat`, `START_SERVER.ps1`, `START_HERE.sh`.
- توثيق: `README.md`, `README2.md`, `README3.md`.

---

3. تفاصيل حسب اللغة/التقنية

HTML

- مسؤول عن هيكل الصفحة والمحتوى الثابت. لتعديل العنوان الظاهر والمتبوعات الاجتماعية (OG) عدّل الـ `<title>` ووسوم `<meta>` في `index.html`.

CSS

- أنماط التصميم في `style.css`، ستجد متغيرات CSS في الأعلى مثل `--primary`، `--accent` والتي تحدد ألوان الثيم الأساسية.

JavaScript

- `script.js` يحتوي على:
  - تعريف الأصناف الأساسية للسلّة وإدارة العناصر.
  - وظائف توليد رسالة الفاتورة والطباعة.
  - الاتصالات مع الـ API عبر `fetch`.
  - سلاسل نصية للمستخدم (تغييرها يؤثر على رسائل التطبيق).

PHP

- الموجود داخل `api/` و`admin/`:
  - `api/db.php` يحتوي على إعدادات الاتصال بقاعدة البيانات ويُنصح بحمايته جيداً (لا تضع كلمات المرور داخل مستودع عام إذا كنت تعمل مع Git العام).
  - نقطة الإدخالات (`order.php`) تتحقق من صحة البيانات قبل الإدخال.

SQL

- `schema.sql` يعرّف الجداول الأساسية. إذا كنت تريد إضافة أعمدة أو جداول جديدة، حدّث هذا الملف ونفّذ التغييرات على قاعدة البيانات عبر migration أو تنفيذ SQL يدويًا.

JSON

- `manifest.json` مهم لتعريف كيفية ظهور التطبيق عند تثبيته على هاتف المستخدم.

---

4. إعداد قاعدة البيانات محلياً (خطوات عملية)

1) افتح XAMPP (أو أي حزمة Apache + MySQL مثبتة) وشغّل Apache وMySQL.
2) افتح `phpMyAdmin` أو استخدم سطر الأوامر لتنفيذ `schema.sql`:

```powershell
mysql -u <db_user> -p < schema.sql
```

3. عدّل إعدادات الاتصال في `api/db.php` بتعبئة:

- `$host = 'localhost';`
- `$dbname = 'myorder';`
- `$user` و`$pass` بما يتطابق مع إعداداتك.

4. اختبر الطلبات عبر واجهة الإدارة أو بواسطة `curl`/Postman إلى `api/order.php`.

---

5. نقاط API الأساسية (موجز)

- `POST api/order.php` — إنشاء طلب جديد. المتوقع: JSON يحتوي على بيانات العميل ومحتويات السلة.
- `GET api/orders.php` — جلب الطلبات (يدعم فلترة الحالة مثلاً).
- `POST api/update_order.php` — تحديث حالة طلب.

مثال سريع لإرسال طلب بواسطة `curl`:

```bash
curl -X POST 'http://localhost/My-Order-main/api/order.php' \
  -H 'Content-Type: application/json' \
  -d '{"customerName":"Ali","customerPhone":"010...","items":[{"id":1,"quantity":2}],"total":200}'
```

---

6. لوحة الإدارة (Admin)

- مواقع الملفات: مجلد `admin/`.
- لتعديل اسم المطعم الافتراضي في لوحة الإعدادات، افتح `admin/dashboard.php` وابحث عن الحقل `restaurantName` وقم بتغيير القيمة الافتراضية.
- لتغيير بيانات الدخول الافتراضية، افتح `api/db.php` أو `api/config.php` إن وُجدت.

نصيحة أمان: استخدم HTTPS، قم بتأمين أمور الجلسة، وغيّر بيانات الاعتماد الافتراضية قبل تشغيل البيئة على الإنترنت.

---

7. PWA وService Worker

- `manifest.json`:
  - غير `name` و`short_name` لأجل تغيير اسم التطبيق عند التثبيت.
  - استبدل مسارات الأيقونات إذا ضفت أيقونات جديدة.

- `sw.js`:
  - لتحديث الكاش، غيّر قيمة `CACHE_NAME` واجعل المتصفح يعيد تحميل الموارد.

مثال: فرض إعادة تحميل الكاش عن طريق تغيير `CACHE_NAME` من `mqt-cache-v1` إلى `mqt-cache-v2`.

---

8. سكربتات التشغيل والتطوير المحلي

- تشغيل سريع (Windows): افتح `START_SERVER.bat` أو `START_SERVER.ps1` إذا كنت تريد سيرفر ملفات ثابتة.
- باستخدام XAMPP: ضع المجلد تحت `c:/xampp/htdocs/` وشغّل Apache وMySQL، ثم افتح `http://localhost/My-Order-main/`.

---

9. أمثلة عملية متقدمة (تغيير اسم الموقع وأيقوناته)

- لتغيير اسم الموقع الذي يظهر في التبويبة والمتصفح:
  1. افتح `index.html` وعدل الوسم `<title>`.
  2. افتح `manifest.json` وعدّل `name` و`short_name`.
  3. داخل `script.js`، ابحث عن أي سلاسل تعرض اسم الموقع (مثل `موقع طلب الطعام الاحترافية`) واستبدلها.

- لتغيير أيقونات PWA:
  1. ضع الصور الجديدة داخل `icons/` (أحجام 192×192 و512×512).
  2. حدّث المسارات في `manifest.json` و`index.html` حيث تُستدعى.

---

10. مشكلات شائعة وحلول سريعة

- خطأ اتصال بقاعدة البيانات: تحقق من بيانات الاعتماد في `api/db.php` ومن تشغيل خدمة MySQL.
- مشكلة CORS عند استدعاءات API من دومين مختلف: أضف رؤوس CORS في ملفات PHP أو استعمل نفس المضيف/المنفذ أثناء التطوير.
- Service Worker لا ينطلِق: امسح كاش المتصفح وأعد تحميل الصفحة، ثم تحقق من Console في DevTools.

---

11. خطوات النشر الموصى بها

1) إعداد خادم Linux/Apache أو Nginx مع PHP وMySQL.
2) استيراد `schema.sql` إلى قاعدة بيانات الإنتاج.
3) تحديث `api/db.php` ببيانات الإنتاج السليمة.
4) إعداد شهادة SSL وتمكين HTTPS.
5) تعطيل أي إعدادات تطوير (مثل تسجيلات الـ console الكثيفة) وحماية ملفات الـ config.

---

12. كيفية المساهمة

- استخدم نظام Git: أنشئ فرعًا جديدًا، أضف تغييراتك، ثم افتح Pull Request مع وصف واضح.
- عند تعديل وظائف في `api/` اختبر أن أي تغييرات تتوافق مع قاعدة البيانات.

---

هل تريد الآن أن أتحوّل إلى أي من الخيارات التالية؟

- توليد نسخة إنجليزية كاملة من هذا الدليل.
- إضافة روابط قابلة للنقر إلى الملفات داخل المستودع في أقسام `ملفات مهمة`.
- استخراج كل النصوص الظاهرة إلى ملف `i18n.json` لترجمة أسهل لاحقًا.

أخبرني أي خيار أبدأ به وسأكمّل العمل.
